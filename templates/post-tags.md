# Справочник тегов

Не копируй этот файл целиком в канал. Нужна **одна** строка хэштегов в конце поста.

## 1. Общая архитектура приложения

Пост про систему целиком: сервисы, потоки данных, интеграции.

```
#architecture #systemdesign #softwarearchitecture #devlog #buildinpublic
```

При необходимости добавь 1–2 тега под стек, например `#typescript` `#kubernetes`.

## 2. Конкретный компонент / слой

Пост про один уровень: backend, API, БД, фронт и т.д.

**База** (часто добавляют):

```
#devlog #buildinpublic
```

**Плюс теги по фокусу** (выбери своё):

| Фокус          | Примеры тегов                          |
|----------------|----------------------------------------|
| Backend        | `#backend #api #serverside`            |
| Frontend       | `#frontend #web #ui`                   |
| API            | `#api #rest #graphql`                  |
| Данные         | `#database #sql #migrations`         |
| Инфраструктура | `#infra #devops #ci #cd`               |
| Мобильный      | `#mobile #ios #android`                |

**Пример только backend:**

```
#backend #api #devlog #buildinpublic
```

**Пример фронт + API:**

```
#frontend #api #devlog #buildinpublic
```

В [`telegram-post-devlog.md`](telegram-post-devlog.md) в конце — один пример строки тегов; замени на нужный из этого файла.
